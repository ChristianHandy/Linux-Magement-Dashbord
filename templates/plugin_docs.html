Overview

FleetPilot plugins are plain Python files placed in the addons/ directory. They are loaded automatically at startup. A plugin can inject HTML into any part of the dashboard, register Flask routes, run background threads, or react to system events (like a completed update run).

✅ What plugins CAN do
  • Add widgets to the dashboard
  • Add buttons to host cards
  • Add tabs to host detail view
  • Add sidebar navigation links
  • React to update events
  • Add Flask routes / pages
  • Run background threads
  • Send external notifications
⚠️ Security notes
  • Plugins run with full app privileges
  • Only install from trusted sources
  • Admin role required to install
  • Restart required after install
  • Never store secrets in plugin code

Plugin Structure

Every plugin is a single .py file with two optional parts:

# my_plugin.py
# ─────────────────────────────────────────────────────────────────────────────
# Part 1: addon_meta — describes the plugin and registers hooks
# ─────────────────────────────────────────────────────────────────────────────
addon_meta = {
    # Required
    "name":        "My Plugin",
    "version":     "1.0.0",
    "author":      "Your Name",
    "description": "Short description shown in the Plugin Manager",
    "scope":       "global",  # global | disks | hosts | dashboard | scanner

    # Optional: hook registrations (see Hook Reference below)
    "dashboard_widget": lambda hosts, stats: "<div>...</div>",
    "sidebar_link":     lambda: {"label": "My Page", "url": "/my_page", "icon": "🔧"},
}

# ─────────────────────────────────────────────────────────────────────────────
# Part 2: register() — called once at startup with the Flask app and disk core
# ─────────────────────────────────────────────────────────────────────────────
def register(app, core):
    """Optional. Use to add routes, start threads, etc."""
    from flask import Blueprint
    bp = Blueprint("my_plugin", __name__)

    @bp.route("/my_page")
    def my_page():
        return "<h1>My Plugin Page</h1>"

    app.register_blueprint(bp)
    print("[my_plugin] loaded")

Adding Flask Routes

def register(app, core):
    from flask import Blueprint, render_template, jsonify, request, session
    from functools import wraps

    bp = Blueprint("my_plugin", __name__, url_prefix="/my_plugin")

    # Re-use the existing login_required decorator
    def login_required(f):
        @wraps(f)
        def decorated(*args, **kwargs):
            if not session.get("user_id"):
                from flask import redirect, url_for
                return redirect(url_for("login"))
            return f(*args, **kwargs)
        return decorated

    @bp.route("/")
    @login_required
    def index():
        return render_template("addons/my_plugin.html")

    @bp.route("/api/data")
    @login_required
    def api_data():
        return jsonify({"status": "ok", "data": []})

    app.register_blueprint(bp)

Security Guidelines

  • Plugins run with full application privileges — treat them like application code, not user content.
  • Only install plugins from sources you trust completely.
  • Never hardcode API keys, passwords, or tokens in plugin files. Use environment variables instead.
  • If your plugin makes outbound HTTP requests, validate all URLs and handle errors gracefully.
  • Sanitize any user input before using it in shell commands or SQL queries.
  • Use Flask's login_required pattern on all routes your plugin adds.
